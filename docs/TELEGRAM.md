# Telegram 手机控制台

## 1. 创建机器人

1. 在 Telegram 中打开 `@BotFather`。
2. 发送 `/newbot`。
3. 设置机器人名称和以 `bot` 结尾的用户名。
4. 保存 BotFather 返回的 Token。

Token 等同机器人密码，不要提交到 Git、发送到群聊或放进截图。

## 2. 面板配置

以管理员登录 Flux Panel Plus，在系统配置中：

1. 填写 `telegram_bot_token`。
2. 启用 `telegram_enabled`。
3. 保存配置。

后端启动或配置重载时会自动调用 `setMyCommands` 和 `setChatMenuButton`，注册 Telegram 输入框旁的 `/` 命令菜单。

## 3. 绑定用户

用户在面板个人设置中生成 Telegram 绑定码，然后私聊机器人：

```text
/bind 绑定码
```

绑定完成后发送：

```text
/menu
```

解绑：

```text
/unbind
```

## 4. 命令

| 命令 | 说明 | 权限 |
|---|---|---|
| `/start` | 使用说明 | 全部绑定用户 |
| `/help` | 命令帮助 | 全部绑定用户 |
| `/menu` | 打开手机控制台 | 全部绑定用户 |
| `/bind` | 绑定面板账号 | 未绑定用户 |
| `/unbind` | 解除绑定 | 已绑定用户 |
| `/login` | 获取面板登录链接 | 全部绑定用户 |
| `/usage` | 查看流量和到期时间 | 全部绑定用户 |
| `/tunnels` | 隧道列表 | 按面板权限 |
| `/new_tunnel` | 新增隧道向导 | 管理员 |
| `/tunnel ID` | 隧道详情 | 按面板权限 |
| `/forwards` | 转发列表 | 按面板权限 |
| `/rules` | 旧版转发列表命令 | 按面板权限 |
| `/new_forward` | 新增转发向导 | 用户与管理员 |
| `/forward ID` | 转发详情 | 资源所有者/管理员 |
| `/nodes` | 节点监控 | 按节点权限 |
| `/node ID` | 节点详情 | 按节点权限 |
| `/audit` | 流量审计 | 用户仅看自己的记录 |
| `/enable ID` | 恢复转发 | 资源所有者/管理员 |
| `/disable ID` | 暂停转发 | 资源所有者/管理员 |
| `/cancel` | 取消新增向导 | 当前用户 |
| `/users` | 用户概览 | 管理员 |
| `/reset ID` | 重置用户流量 | 管理员 |

## 5. 新增隧道

发送 `/new_tunnel` 或在“隧道管理”页面点击“新增”：

1. 选择单节点端口转发或双节点隧道。
2. 选择入口节点。
3. 双节点隧道继续选择出口节点。
4. 选择协议。
5. 输入名称和可选网卡。
6. 检查摘要并确认创建。

只有管理员可以新增隧道。

## 6. 新增转发

发送 `/new_forward` 或在“转发管理”页面点击“新增”：

1. 选择隧道；管理员还可以选择路由。
2. 输入转发名称。
3. 输入目标，例如 `1.2.3.4:8080`；多个目标可换行或用逗号分隔。
4. 自动分配或手动指定入口端口。
5. 选择轮询、随机、故障转移或哈希策略。
6. 设置监听地址和可选出口网卡。
7. 检查摘要并确认创建。

向导 15 分钟无操作会过期，随时可以发送 `/cancel`。

## 7. 后台诊断与性能

- 隧道和转发诊断会发送独立进度消息，原详情卡片和诊断按钮保持不变。
- 同一聊天、同一资源、同一诊断类型同时只执行一个任务。
- Callback 接收与页面渲染由 16 个 Worker 处理，并使用 64 条缓冲队列。
- `answerCallbackQuery` 异步执行，避免确认请求阻塞页面数据。
- Telegram HTTP 客户端启用 Keep-Alive、HTTP/2 和空闲连接池。

## 8. 权限模型

- 每次 Callback 都会重新按 chat ID 查询绑定用户。
- 普通用户只能查看和操作自己的转发、获授权的隧道与节点。
- 普通用户审计查询强制附加自己的 `user_id`。
- 管理员命令即使显示在普通用户的 `/` 菜单中，后端仍会拒绝越权执行。
- Callback Data 不包含 Token、密码或节点 Secret。

## 9. 排障

```bash
docker logs --tail=200 go-backend | grep Telegram
curl -I https://api.telegram.org
```

如果没有 `/` 菜单：

1. 重新进入机器人私聊。
2. 发送一次 `/start`。
3. 检查后端是否成功启动 Telegram。
4. 确认 Bot Token 正确且服务器可以访问 Telegram API。

如果所有按钮都慢，重点检查 VPS 到 Telegram API 的丢包、DNS、代理和 TLS 握手耗时。
